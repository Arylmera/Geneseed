# Web UI v2 — navigation shell, doctor, selective restore, themes, offline package, graph

**Date:** 2026-06-12
**Status:** implemented (verified 2026-06-15)

## Problem

The first web UI iteration (spec `docs/superpowers/specs/2026-06-12-web-ui-design.md`)
shipped a dashboard, a section browser, a diff/export page, and a console for three
fire-and-notify actions. It mirrors the TUI but adds little over it:

- **Doctor** dumps raw text into the console; the dashboard badge says "N issues"
  with no way to see *which check* failed or what to do about it.
- **The diff page is read-only** — the only remedy for drift is the all-or-nothing
  `update` action. There is no way to restore a single file from source.
- **Themes are invisible.** 14 themes exist and the server already validates a
  `{theme}` override on the build action, but the UI exposes none of it.
- **Corporate (offline) installs** update by manual zip (no GitHub access), and
  nothing produces that zip. `upgrade` only knows how to download from GitHub.
- **Cross-links** between agents/skills are resolved per-item but there is no
  overview — orphans and hubs are invisible.
- **Navigation is flat**: one header row, no place for new pages, and no view that
  explains how the local harness is actually set up (target dir, emit mode, theme,
  deployed state).

## Fix

Six self-contained increments, one commit each, in this order. Every frontend
commit rebuilds the committed `web/dist` so each commit stays deployable.

### 1. Navigation shell + Settings page (the frame the rest lands in)

- **Sidebar navigation** replaces the header-button row: Dashboard, Library
  (agents/skills/laws/memory/notebook/config), Changes (diff), Doctor, Themes,
  Graph, Settings. Active page highlighted; console drawer unchanged.
- **Routes added:** `#/doctor`, `#/themes`, `#/graph`, `#/settings`.
- **Settings page** = "how is my harness set up": target directory, emit mode,
  theme, deployed yes/no, last build time, counts, server port — read from the
  existing `/api/overview` plus a small `GET /api/setup` (adds geneseed version
  from `_update.py`'s pinned ref if available, source root path, python version).
  Actions that belong to setup live here too: Update, Build, and (increment 5)
  the offline package download.
- **Dashboard becomes the entry point**: status hero (deployed/theme/emit at a
  glance), doctor + drift summary cards linking to their pages, section count
  cards linking into Library.
- **UI design pass** across `styles.css`: consistent spacing/typography scale,
  card and badge tokens, accent derived from the deployed theme's `ACCENT`,
  sidebar layout responsive down to narrow windows.

### 2. Doctor page — structured report

- `harness._doctor_collect` already runs five named checks (build scan, theme
  parity, authoring gates, rendered-bundle drift, global emit). Extend it (or a
  thin sibling) to expose **grouped** results: `[{check, label, problems: [...]}]`
  without changing the existing `(themes, problems)` contract used by the CLI.
- New `GET /api/doctor` in `rituals/web.py` returns the grouped shape plus the
  theme list validated.
- **Doctor page** renders one card per check: green "clean" or red with the
  problem list expandable. A "Re-run" button posts the existing `doctor` action
  (raw output still streams to the console); the page refetches when the job ends.
- Dashboard doctor card navigates to `#/doctor`.

### 3. Selective restore on the Changes page

- `_diff_collect` renders the expected copy via `build.emit_opencode_global`
  into a temp dir and compares per `rel`. **Restore reuses exactly that render**:
  new pure function `api_restore(state, files: list[str])` in `web.py` renders
  expected once, then per selected rel:
  - `edited` / `missing` → write the expected content over/into the deployed tree;
  - `added` → delete the deployed file (UI requires explicit confirmation,
    deletion is irreversible).
- Token-guarded `POST /api/actions/restore` with body `{files: [rel, ...]}`.
  Synchronous (the render is the same cost as one diff GET) — returns
  `{restored: [...], deleted: [...], errors: [...]}`, then the UI reloads the diff
  and the overview. Unknown rels are reported in `errors`, never written.
- **Changes page** gains per-file checkboxes, select-all per status group, and a
  "Restore selected from source" button. Direction is explicit in the UI copy:
  *source wins, local edits are discarded* (the inverse — keeping local edits —
  remains Export improvements).

### 4. Theme gallery

- Enrich `api_themes` with per-theme `accent`, `tagline`, and `sigil` read from
  `themes/*.json` (`ACCENT`, `TAGLINE`, `LOADED_SIGIL`).
- **Themes page**: one card per theme — accent swatch, name, blurb, sample voice
  line; the deployed theme highlighted. "Apply" posts the existing `build`
  action with `{theme}` (server already validates overrides); the rebuild
  streams in the console and the overview refreshes when it lands.
- `api.action(name, body)` gains an optional JSON body (today it always sends `{}`).

### 5. Offline update package

- `GET /api/offline-zip` streams a zip of the source tree: `git archive HEAD`
  when git is available, else a `zipfile` walk honouring a small exclusion list
  (`.git`, `node_modules`, `web/node_modules`, `__pycache__`). Download name
  `geneseed-offline-<YYYYMMDD>.zip`. GET (read-only), localhost-only like every
  other read.
- **Consume side:** `upgrade --zip <file>` in `rituals/_update.py` feeds the
  existing extract-and-refresh path from a local zip instead of the GitHub
  download. Same factory-file refresh, same theme/emit preservation.
- Settings page gets a "Download offline package" button with a short
  explanation of the corporate use case.

### 6. Cross-link graph

- New `GET /api/graph`: nodes = agents + skills (`{id, type, name}`), edges from
  the existing `WIKILINK_RE` resolver applied to every body
  (`{source, target}`, deduplicated). Pure function `api_graph(state)`.
- **Graph page**: dependency-free SVG force layout (~60 lines hand-rolled
  simulation, no library), agents vs skills colour-coded, orphan nodes dimmed,
  click navigates to `#/item/<type>/<name>`, hover highlights the neighbourhood.

## Testing

- Python: unit tests in `tests/test_web.py` for `api_doctor` (grouped shape),
  `api_restore` (restores edited, re-creates missing, deletes added,
  rejects unknown rels — deletion confirmation is a UI concern), enriched `api_themes`, `api_graph` (edges resolve, no
  self-links), and the offline-zip content listing. `upgrade --zip` covered in
  `tests/test_update.py`.
- JS: extend the dashboard smoke test for the sidebar shell; smoke tests for the
  Doctor and Themes pages (vitest, jsdom, mocked `api`).

## Out of scope

- Persistent job history across server restarts (declined in this round).
- Editing agent/skill bodies from the browser.
- Any network dependency or npm runtime dependency beyond the existing React.
