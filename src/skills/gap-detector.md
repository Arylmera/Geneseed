# {{SKILL}}: gap-detector

> {{DESC_GAP_DETECTOR}}

**Trigger:** the user believes they already master a skill or topic and wants that belief stress-tested — "I think I already know X, prove me wrong", "test whether I really understand X", "find the holes in what I know". For routing toward a goal use [learning-path](learning-path.md); for checking freshly-studied material use [feynman](feynman.md).

## Procedure
1. Take the skill the user claims to master. Set the contract plainly: you will try to prove them wrong, and you will not go easy.
2. Ask five questions that look simple but separate someone who has truly gone deep from someone who has only skimmed — edge cases, "why", the assumption under the rule, the case where the standard answer breaks.
3. For every answer the user gives, say plainly what it reveals about what is still missing in their foundation. If an answer is shallow, name it as shallow and say what depth would have looked like — no flattery.
4. Close with the honest verdict: the specific gaps to fill before the mastery claim holds.

## Done when
- All five questions were asked, each answer was met with a direct read of what it exposes, and the run ends with a concrete verdict on the remaining gaps — never empty praise.
