# {{SKILL}}: prose

> {{DESC_PROSE}}

**Trigger:** about to hand a human written prose — a commit body, PR description,
doc, code comment, {{MEMORY}} note, or a written reply — and it should read like a
person wrote it, not a model.

## Procedure
1. Draft for content first, fix voice on a second pass — don't self-censor while
   the idea is still forming.
2. Cut throat-clearing and filler: openers like "In today's world", "It's worth
   noting", "I'll help you", and hedges that carry no information.
3. Active voice, human subject. Don't let inanimate things perform human actions
   ("the test wants…", "the code believes…").
4. Be concrete: swap vague abstractions and intensifiers for the specific fact,
   number, or name. Drop adverbs propping up a weak verb ({{LAW}} XV — every word
   earns its place).
5. Vary rhythm: don't run three same-length sentences back to back, and break the
   reflex "X, Y, and Z" triad when it shows up in every paragraph.
6. Trust the reader: cut restatements, summaries of what you just said, and
   quotable-sounding pronouncements. Remove decorative em-dashes.
7. Read it once as the recipient. Rate 1–10 on directness, rhythm, reader-trust,
   authentic voice, and density (how much is still cuttable); under ~35/50, cut
   and revise before sending.

## Done when
- The text says what it means in the fewest honest words, carries no AI tells,
  and would pass as written by a careful human.

## Self-improvement

Close each run with one beat of reflection on the {{SKILL}} itself:
- A step misled, a needed step was missing, or the trigger fired wrongly — that
  is a flaw in this file. Propose the exact edit (trigger, procedure, or
  done-when) and apply it with the user's assent ({{LAW}} II).
- A lesson that is *not* a flaw in this file goes to {{MEMORY}} only if it
  clears {{LAW}} VI's bar: it would change how a future session behaves, and a
  fresh read of the repo would not re-derive it. Update an existing memory over
  adding one; when in doubt, leave it out.
- No friction, nothing learned — move on; this loop earns no ceremony. Most
  runs end here.
